## 两种测试命名方法

首先来看下我们的第一个测试，将其方法名称修改为以下：

*tests/Feature/ViewQuestionsTest.php*

```
<?php

namespace Tests\Feature;

use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class ViewQuestionsTest extends TestCase
{

    /** @test */
    public function user_can_view_questions()
    {
        // 0. 抛出异常
        $this->withoutExceptionHandling();

        // 1. 访问链接 questions
        $test = $this->get('/questions');

        // 2. 正常返回 200
        $test->assertStatus(200);
    }
}

```

运行测试：

![编写和运行测试](https://cdn.learnku.com/uploads/images/202005/30/19192/ZqnN0y9bg5.png!large)

可以看到测试成功通过。

通常而言，编写一个测试一般有两种方式：

1. 方法名以 `test` 开头，其后为驼峰，如`testUserCanViewQuestions()`;
2. 以 ` /** @test */` 标识测试，方法名为下划线风格；

**一般推荐使用第 2 种**，因其具有更佳的可读性，因此本教程中所有测试方法均为此风格。
