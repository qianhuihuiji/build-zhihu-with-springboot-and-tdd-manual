一般而言，我们经常会用三种方式运行测试：

- 运行所有测试
- 运行单个文件中的所有测试
- 运行单个测试

通常在推送代码之前，或者是验证某个功能是否影响到系统其它功能时，我们需要用以下命令来运行所有测试：

```
$ phpunit
```

效果如下：

![编写和运行测试](https://cdn.learnku.com/uploads/images/202005/30/19192/hnedUqGBHl.png!large)

但是大多时候，我们只会单独运行某个文件里的测试或者某一个测试，这样可以是我们的测试进度更快，问题更快暴露出来。我们使用以下命令测试的单个文件：

```
$ phpunit tests/Feature/ViewQuestionsTest.php 
```

> 注：`phpunit` 后面是要测试的文件在 `test` 目录下的路径

效果如下：

![编写和运行测试](https://cdn.learnku.com/uploads/images/202005/30/19192/nCHT8drmDO.png!large)

还能直接使用`--filter` 参数的方式直接运行单个测试：

```
$ phpunit --filter user_can_view_questions
```

> 注：`--filter` 后面是测试的方法名
