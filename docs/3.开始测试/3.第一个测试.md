## 测试文件

在新创建的项目中，`Laravel` 为我们提供了一个测试样例文件，`tests/Feature/ExampleTest.php`，我们将其重命名为：
`ViewQuestionsTest.php` 并书写第一个测试：

```
<?php

namespace Tests\Feature;

use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class ViewQuestionsTest extends TestCase
{
    public function testUserCanViewQuestions()
    {
		// 1.假设 /questions 路由存在
        // 2. 访问链接 /questions
        // 3. 正常返回 200
    }
}
```

通常我们组织一个测试，需要分三步走：前提条件、行为动作以及预期结果。那上面这个测试来举例，分别对应：

- 前提条件：假设已存在路由
- 行为动作：然后我们访问路由
- 预期结果：我们要看到预期结果

我们在上面的代码写上思路，现在补充代码：

```
<?php

namespace Tests\Feature;

use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class ViewQuestionsTest extends TestCase
{
    public function testUserCanViewQuestions()
    {
		// 1.假设 /questions 路由存在
		// 2. 访问链接 questions 
        $test = $this->get('/questions');

		// 3. 正常返回 200
        $test->assertStatus(200);
    }
}

```

接下来我们要先运行测试：

```
$ phpunit 
```
